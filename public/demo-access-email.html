2) How to sign in (important)
A. Open the Frontend → click Dashboard Login
B. Enter email + password for a test account
C. Requester / Approver / Processor: after password is accepted, the app sends a 6-digit verification code to that user’s email. Enter the code to finish sign-in (2FA). Use Resend if needed.
D. Admin: password only (no email OTP by default). Admin can optionally enable an Authenticator app later in Profile.
E. You land on /dashboard/ (home). Use sidebar for your role pages.
F. Always Sign Out before switching to another role account.
3) Complete business flow (do this end-to-end)

This is the proper path that exercises every major function.

FLOW A — Happy path with partial payment
A1
Requester creates expense
Login as Requester → Submit Expense
Fill: Country (auto currency) · Project · Category · Local amount · Expense date · Due date · Description · Invoice file
Submit → system stores USD amount + FX rate + date
Status becomes Pending Approver
Check My Requests and confirmation email
A2
Approver first review — Request Changes
Sign out → Login as Approver (password + email OTP)
Open Approver queue → open the request → View details
Confirm you see: USD amount, local amount, FX rate, invoice preview/download, description, due date, history
Choose one action:
Action Result
Approve Moves to Processor queue (Pending Processing). Notes saved.
Reject Ends workflow (reason required). Requester notified.
Request Changes Status → Changes Requested. Command/notes saved. Requester must edit & resubmit.
Try this: use Request Changes once (e.g. “Please update description / amount”), then continue the loop below.
A3
Requester edits & resubmits (only when Changes Requested)
Login as Requester → My Requests
Find status Changes Requested → use Edit (Edit is available only in this status)
Update description/amount/invoice as asked → save/resubmit
History adds: Request Details Modified + Resubmitted after Changes
Status returns to Pending Approver · Approver reviews again and can Approve
(Requester cannot delete requests. Staff can delete when permitted.)
A4
Approver approves
Login as Approver → open request → Approve (add notes, e.g. “ok”)
Status → Pending Processing (ready for finance)
Processor receives email notification
A5
Processor optional return (test control)
Login as Processor (password + email OTP) → Processor queue
Optional test: Request Changes
  • Back to Requester → Changes Requested again
  • Back to Approver → returns to Approver queue for re-approval
Then bring it back to approved/pending processing before payment
A6
Processor payment (full or partial)
Login as Processor (password + email OTP) → Processor queue. Options:
Action What to try
Partial Pay Amount < total + receipt + notes → Partially Paid. Remaining balance shown.
Mark Paid / Process Pay remaining (or full) + receipt → Processed & Paid.
Reject Payout Reject with reason; requester notified.
Request Changes Send back to Requester or Back to Approver with command/notes.
Suggested: Partial pay first + receipt → then pay remaining + second receipt. Details should show Total Paid, Remaining $0.00, both receipts, payment history.
A7
Processor final payment
Pay remaining amount · upload second receipt · notes (e.g. “done”)
Status → Processed & Paid · Remaining $0.00
Requester receives “paid” email · both receipts visible in details
FLOW B — Quick reject path (optional second test)
Submit another expense as Requester → Approver Reject (reason required) → status Rejected by Approver · requester notified.
Or approve then Processor Reject Payout → Rejected by Processor.
